# Placeholders

OreoEssentials registers two PlaceholderAPI expansions:

| Identifier | Used for |
|---|---|
| `oreo` | All core module placeholders |
| `oreoah` | Auction House placeholders |

PlaceholderAPI must be installed for these to work. Both expansions register with `persist = true` and survive `/papi reload` without restarting the server.

---

## Economy

| Placeholder | Returns | Notes |
|---|---|---|
| `%oreo_balance%` | Raw double (e.g. `12345.67`) | Returns `0` if economy is unavailable |
| `%oreo_balance_formatted%` | Formatted with commas (e.g. `12,345.67`) | Uses `#,##0.##` pattern; returns `0` if unavailable |

## Kits

### Global Kit Info

| Placeholder | Returns |
|---|---|
| `%oreo_kits_enabled%` | `true` or `false` — whether the kits module is enabled |
| `%oreo_kits_count%` | Number of kits defined in config |

### Per-Player Kit Info (requires online player)

| Placeholder | Returns |
|---|---|
| `%oreo_kits_ready_count%` | How many kits this player can claim right now |
| `%oreo_kits_ready_list%` | Comma-separated list of ready kit IDs (trimmed to 64 chars) |

A kit is counted as "ready" if:
- The player has the `oreo.kit.claim` permission
- The kit cooldown has expired **or** the player has `oreo.kit.bypasscooldown`

### Per-Kit Dynamic Placeholders

Replace `<kitId>` with the exact kit ID (lowercase).

| Placeholder | Returns |
|---|---|
| `%oreo_kit_ready_<kitId>%` | `true` if the player can claim this kit right now, `false` otherwise |
| `%oreo_kit_cooldown_<kitId>%` | `ready` if no cooldown, or remaining seconds as a number |
| `%oreo_kit_cooldown_formatted_<kitId>%` | `ready` if no cooldown, or a human-readable duration (e.g. `1d 2h 3m` / `45s`) |

**Example:**
```
%oreo_kit_ready_starter%       → true
%oreo_kit_cooldown_daily%      → 3600
%oreo_kit_cooldown_formatted_daily%  → 1h
```

## Playtime

Playtime is resolved in priority order: PlaytimeRewardsService (if enabled) → PlaytimeTracker → Bukkit PLAY_ONE_MINUTE statistic.

### Base Playtime

| Placeholder | Returns |
|---|---|
| `%oreo_playtime_total_seconds%` | Total playtime in whole seconds |
| `%oreo_playtime_total_minutes%` | Total playtime in whole minutes (seconds ÷ 60) |
| `%oreo_playtime_total_hours%` | Total playtime in whole hours (seconds ÷ 3600) |
| `%oreo_playtime_total_days%` | Total playtime in whole days (seconds ÷ 86400) |

All values are floored integers, not rounded.

### Playtime Rewards

| Placeholder | Returns |
|---|---|
| `%oreo_prewards_enabled%` | `true` or `false` — whether the playtime rewards system is enabled |
| `%oreo_prewards_ready_count%` | Number of rewards this player can claim right now |
| `%oreo_prewards_ready_list%` | Comma-separated list of claimable reward IDs (trimmed to 64 chars) |

### Per-Reward State (replace `<rewardId>` with the reward's ID)

| Placeholder | Returns |
|---|---|
| `%oreo_prewards_state_<rewardId>%` | One of: `LOCKED`, `READY`, `CLAIMED`, `REPEATING` |

**State meanings:**
- `LOCKED` — player hasn't reached the required playtime yet
- `READY` — player meets the requirement and hasn't claimed it
- `CLAIMED` — player already claimed this one-time reward
- `REPEATING` — reward is repeatable and the player has claimed at least one cycle

## Player Vaults

### Global Vault System

| Placeholder | Returns |
|---|---|
| `%oreo_vaults_enabled%` | `true` or `false` — whether the PlayerVaults module is enabled |
| `%oreo_vaults_max%` | Maximum number of vaults configured (global cap) |

### Per-Player Vault Counts (requires online player)

| Placeholder | Returns |
|---|---|
| `%oreo_vaults_unlocked_count%` | Number of vaults this player can currently access |
| `%oreo_vaults_unlocked_list%` | Comma-separated list of accessible vault numbers (trimmed to 64 chars) |
| `%oreo_vaults_locked_list%` | Comma-separated list of locked vault numbers (trimmed to 64 chars) |

### Per-Vault Dynamic Placeholders

Replace `<n>` with the vault number (e.g. `1`, `2`, `3`).

| Placeholder | Returns |
|---|---|
| `%oreo_vault_can_access_<n>%` | `true` if the player can open vault `<n>`, `false` otherwise |
| `%oreo_vault_slots_<n>%` | Number of item slots in vault `<n>` for this player (permission-resolved) |
| `%oreo_vault_rows_<n>%` | Number of rows in vault `<n>` (slots ÷ 9, minimum 1) |
| `%oreo_vault_title_preview_<n>%` | The colored title of vault `<n>` with `<id>` and `<rows>` already replaced |

**Example:**
```
%oreo_vault_can_access_3%      → true
%oreo_vault_slots_1%           → 27
%oreo_vault_rows_1%            → 3
%oreo_vault_title_preview_2%   → §6Vault 2 (3 rows)
```

## Homes

| Placeholder | Returns |
|---|---|
| `%oreo_homes_used%` | Number of homes this player currently has set |
| `%oreo_homes_max%` | Maximum homes allowed for this player (permission-based) |
| `%oreo_homes%` | Shorthand: `used/max` (e.g. `3/5`) |

## TempFly

These placeholders require the TempFly module. All return fallback values if the module is not loaded or the player is offline.

| Placeholder | Returns |
|---|---|
| `%oreo_tempfly_active%` | `true` if the player has active TempFly time remaining, `false` otherwise |
| `%oreo_tempfly_seconds%` | Remaining TempFly time in whole seconds (returns `0` if inactive) |
| `%oreo_tempfly_time%` | Formatted remaining time (e.g. `5m 30s`, `2h 15m`) or `inactive` if none |
| `%oreo_tempfly_time_or_empty%` | Same as above but returns an empty string instead of `inactive` |

**Format rules for `tempfly_time`:**
- Under 60 seconds: `<s>s` (e.g. `45s`)
- Under 1 hour: `<m>m <s>s` or `<m>m` if seconds are zero
- 1 hour or more: `<h>h <m>m` or `<h>h` if minutes are zero

## Server & Network

### This Server

| Placeholder | Returns |
|---|---|
| `%oreo_server_name%` | The configured server ID (from OreoEssentials core config) |
| `%oreo_server_nick%` | The display nickname of this server (from `server_nicknames.map` in config), falls back to raw server name |
| `%oreo_server_players_total%` | Online player count on this server (uses RabbitMQ cache if available, falls back to Bukkit) |
| `%oreo_network_online%` | Local Bukkit online player count (always this server only, no cache) |

### Other Servers

| Placeholder | Returns |
|---|---|
| `%oreo_server_nick_<serverId>%` | Display nickname for any server by its ID |
| `%oreo_server_players_<serverName>%` | Player count on a specific named server (from RabbitMQ network cache) |
| `%oreo_<serverName>_players_total%` | Alternate format — player count on a specific server |
| `%oreo_crossserver_players_total%` | Total players across ALL servers in the network (sum from RabbitMQ cache) |

**Example:**
```
%oreo_server_nick_lobby%            → Lobby
%oreo_server_players_survival%      → 23
%oreo_survival_players_total%       → 23
%oreo_crossserver_players_total%    → 87
```

> `server_players_<name>` and `<name>_players_total` do the same thing — both are supported for format flexibility. Use whichever fits your scoreboard layout better.

### World

| Placeholder | Returns |
|---|---|
| `%oreo_world_name%` | The player's current world name |
| `%oreo_world%` | Alias for `world_name` |
| `%oreo_world_nick%` | Display nickname of the player's current world (from `world_nicknames.map` in config) |
| `%oreo_world_nick_<worldId>%` | Display nickname for any world by its name |

**Configuring server and world nicknames** (`config.yml`):
```yaml
server_nicknames:
  default: Unknown
  map:
    lobby: "Lobby"
    survival: "Survival"
    skyblock: "SkyBlock"

world_nicknames:
  default: Unknown
  map:
    world: "Overworld"
    world_nether: "Nether"
    world_the_end: "The End"
```

## Auction House

These use the **`oreoah`** identifier (not `oreo`). Requires PlaceholderAPI and the Auction House module to be enabled.

| Placeholder | Returns |
|---|---|
| `%oreoah_active_listings%` | Total number of active listings across all players on this server |
| `%oreoah_player_listings%` | Number of active listings by this player |
| `%oreoah_player_sales%` | Number of items this player has sold (sold history count) |
| `%oreoah_player_expired%` | Number of expired listings this player has not yet reclaimed |

> In cross-server mode, `active_listings` reflects all listings loaded into this server's in-memory cache — which is the full network listing set when MongoDB + RabbitMQ sync is working.

---

## Debug Mode

If placeholders are returning unexpected values, enable debug logging in `config.yml`:

```yaml
placeholder-debug: true
```

Every placeholder request will be logged to console with the player name and resolved value. Disable this in production — it is very verbose.
